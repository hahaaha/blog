spring mvc初入门

Posted on 2019-01-10

在了解spring mvc的工作原理之前我们需要对model/view/Controller的概念有个初步的认识

  • model：封装了应用程序数据，且通常由pojo组成
  • view：用来呈现模型数据
  • controller：用来处理用户请求，并构建合适的model，然后传递到view中

    Spring mvc的原理

    Spring mvc的所有操作是围绕DispatcherServlet来进行的。在spring mvc中 DispatcherServlet处理所有的http请求和响应。 DispatcherServlet的工作流程见下图

DispatcherServlet的工作流程：

  1. 接受到一个http请求后，DispatcherServlet根据HandlerMapping来选择适当的控制器
  2. 控制器接受请求后，用Get或Post方法来调用适当的service方法，并会根据业务逻辑定义数据模型，并返回视图名称到DispatcherServlet中
  3. DispatcherServlet会利用ViewResolver（视图解析器）给请求选取视图
  4. 选定好视图后，DispatcherServelet 将把数据模型传递给视图，最后呈现在浏览器中

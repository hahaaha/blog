java io系列（一）文件的写入和读取

Posted on 2019-02-13

java.io主要用途是，系统上的文件和目录的创建以及文件的内容的读取和写入。

java io分为两种模式，一种是字节流，一种是字符流。但是字节流使用的更广泛一些，这是因为所有的文件在硬盘或者传输中都是用字节来进行传输的。

字节流

在字节流中，java使用FileInputStream和FileOutputStream来进行输入流和输出流。

下面是FileInputStream的方法以及方法的描述

方法及描述
public int read(int r)throws IOException{}
这个方法从 InputStream对象读取指定字节的数据。返回为整数值。返回下一字节数据，如果已经到结尾则返回-1。
public int read(byte[] r) throws IOException{}
这个方法从输入流读取r.length长度的字节。返回读取的字节数。如果是文件结尾则返回-1。
public void close() throws IOException{}
关闭此文件输入流并释放与此流有关的所有系统资源。抛出IOException异常。
public int available() throws IOException{}
返回下一次对此输入流调用的方法可以不受阻塞地从此输入流读取的字节数。返回一个整数值。

下面是FileOutputStream的方法和描述

方法及描述
public void write(int w)throws IOException{}
这个方法把指定的字节写到输出流中。
public void write(byte[] w)
把指定数组中w.length长度的字节写到OutputStream中。
protected void finalize()throws IOException {}
这个方法清除与该文件的连接。确保在不再引用文件输入流时调用其 close 方法。抛出IOException异常。
public void close() throws IOException{}
关闭此文件输入流并释放与此流有关的所有系统资源。抛出IOException异常。

字符流

使用字符流时，并不是直接将内容直接写入文件中，而是先将字符写入缓存区中，然后再将缓存区中的内容转换为字节写入文件中。

需要注意的是，FileWriter和FileReader分别是从OutputStreamWriter、InputStreamReader类继承而来。还有一点需要注意的是，FileWrite由于使用字符流的原因，需要使用flush()或者close()来将缓冲区中的内容放入文件中。

下面是FileWrite的方法和描述

方法及描述
public void write(int c) throws IOException
写入单个字符c。
public void write(byte[] w)
把指定数组中w.length长度的字节写到OutputStream中。
public void write(char [] c, int offset, int len)
写入字符数组中开始为offset长度为len的某一部分。
public void write(String s, int offset, int len)
写入字符串中开始为offset长度为len的某一部分。
public void flush()
刷新缓存区，将缓存区内容写入文件中，但不关闭流
public void close()
首先刷新缓存区，然后关闭流
